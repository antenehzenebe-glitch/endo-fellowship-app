## Collaboration & Code Review

### Code Review Standards
- **TypeScript:** All code must pass strict type checking
- **Mobile:** Responsive design must be verified on 320px+ screens
- **RLS:** Data access changes require security review
- **ACGME:** Requirements changes need domain expert sign-off
- **Performance:** Dashboard queries must execute in <2s

### Pull Request Checklist
- [ ] TypeScript types are explicit and validated
- [ ] Mobile responsiveness tested
- [ ] RLS policies verified (if data access changed)
- [ ] ACGME requirements documented (if adding features)
- [ ] Tests pass locally and in CI/CD
- [ ] Code review from appropriate expert(s)

### Escalation Path
1. **Technical questions** → Lead Developer / Tech Lead
2. **ACGME/compliance questions** → QA & Compliance Lead
3. **Database/performance questions** → Backend Developer / Database Specialist
4. **Architecture decisions** → Technical Lead + team discussion
